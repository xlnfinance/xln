# Archive

This directory keeps historical XLN material that still has one of these values:

- shows the evolution of a core idea
- preserves a detailed older plan that was later collapsed into a live doc
- captures historical operational/debug context that may matter again
- preserves positioning/philosophy text that still has narrative value

## What does not belong here

These are now candidates for deletion rather than archival:

- empty templates
- one-off LLM prompts
- closed session notes that are fully absorbed by stronger docs
- stale duplicate status lists with no unique content

## Recommended reading order

Do not start in `archive/`.

Start with live docs:

1. `../readme.md`
2. `../constraints.md`
3. `../status.md`
4. `../mainnet.md`
5. the relevant protocol/spec docs

Use archive only when you need:

- historical wording
- design evolution
- a past implementation plan that still contains useful detail
