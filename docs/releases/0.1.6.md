# xln 0.1.6

## Release Notes

This release establishes the current product and engineering baseline. It
includes all completed work since `0.1.5`, then introduces reproducible release
snapshots so later releases can be judged against raw history instead of memory.

### Runtime Projection

- Unified browser and remote runtime reads behind the same runtime adapter.
- Historical navigation now projects the complete runtime view rather than
  combining old frames with live accounts.
- Added bounded frame caches, explicit return-to-live behavior, and regression
  coverage for local and remote time travel.
- Added deterministic dispute-finalization helpers and focused tests.

### Developer and Product Surfaces

- Added the network graph timeline, runtime diagnostics, entity audit, and
  jurisdiction inspection surfaces.
- Added the RCPAN bilateral dispute microscope and court-ledger explainer.
- Added the xln assistant proxy, contextual guide, and mascot interface.
- Expanded visual and browser coverage around runtime projection and the new
  developer-facing surfaces.

### Release Observability

- Added full repository snapshots with per-file LOC, complexity, hashes,
  dependencies, dependents, test relations, debt markers, and historical deltas.
- Generated and release artifacts are retained in raw exclusion metadata but do
  not inflate the auditable code tree.
- Added the `/releases` history chart with repository and module selectors.
- Canonicalized the project version as `0.1.6` across `VERSION` and package
  metadata.

### Measured Change

- Auditable code: `277,017 -> 393,784` LOC.
- Complexity: `35,501 -> 51,416`.
- Test/source ratio: `19.3% -> 31.6%`.
- Solidity contracts stayed effectively flat: `7,019 -> 6,974` LOC.

The absolute code and complexity increase is now visible and intentionally
becomes the reduction target for the next releases.
